新手最容易死在**同时做 5 个渠道**：每个都做一点、每个都半途而废。这一小时只做一件事——**选定一个主渠道**，做出 30 天的具体动作。

## 🎯 你将完成

- 选定**一个主渠道**（不是 5 个）
- 一份 **30 天动作日历**（每周一个目标）
- 第一批用户的**触达脚本**（社区帖、广告创意、KOL 私信）
- 通过 / 失败的**判断标准**

## 📋 前置要求

- 上一篇 [App Store 上架](/zh/tutorials/app-store-aso/) 的产出：已提交审核或已上架
- 一个能发广告 / 发帖的账号（X、Reddit、抖音、小红书任一）
- $50–200 的冷启动预算（可选但加速）

## 第 1 步：对照你的方向选主渠道（0–10 分钟）

### 三种起步方式

| 方式 | 适合什么方向 | 周期 | 难点 |
|---|---|---|---|
| **ASO** | 工具类，用户主动搜索 | 4–8 周看效果 | 关键词竞争 |
| **UGC（TikTok/Reels/抖音/小红书）** | 能用视觉展示效果的 | 1–2 周可爆 | 持续产内容 |
| **付费广告（Meta/Google）** | 有清晰漏斗，能算 ROI | 1 周见数据 | 烧钱测素材 |

### 怎么选

回答 3 个问题：

1. **用户会主动搜你解决的问题吗？** → 是 → ASO
2. **效果能用 15 秒视频展示吗？** → 是 → UGC
3. **你能算出每个用户 LTV ≥ $5 吗？** → 是 → 付费广告

**第一次做 → 优先选 UGC**。原因：

- 不烧钱
- 见效快（一条视频火了就有第一批）
- 内容产出本身在练手感

### 不要这样做

- ❌ "我朋友在 Reddit 发了帖，我也去发" → 渠道要对应你的方向
- ❌ "我都做"，3 周后哪个都没起来
- ❌ 选 KOL 合作做主渠道（适合做辅助，不适合主力）

## 第 2 步：拆 30 天动作清单（10–25 分钟）

### 通用 30 天模板

```
Week 1 - 触达 5 个垂直社区，只求访谈和反馈
Week 2 - 做 10 条短视频/帖子脚本，连续发 7 天
Week 3 - 小额预算测试 3 组创意（如做付费）
Week 4 - 把胜出角度反过来改截图、onboarding、付费墙
```

### 三种主渠道的具体动作

**ASO 主渠道**：

```
W1: 关键词矩阵更新，每天加 1 条 review，找 5 个朋友给五星
W2: 优化截图（A/B test 第 1 张），副标题改一版
W3: 用 Search Ads 跑 $30 测关键词，看哪个词带来下载最便宜
W4: 把最便宜的词放到 App 名称里，重新提交
```

**UGC 主渠道**：

```
W1: 看竞品高赞视频的前 3 秒钩子，记 20 条
W2: 用同样钩子模板拍 7 条短视频，分散在 7 天发
W3: 看哪条数据最好，拍 5 条仿那条
W4: 在评论区里互动 + 把视频里的 CTA 改进
```

**付费广告主渠道**：

```
W1: 准备 3 组广告创意（不同钩子），同一个落地页
W2: 每组每天 $5，跑 5 天，看 CTR + CPI
W3: 砍掉最差的，把预算加到最好的那组
W4: 找出胜出创意，套用到 App Store 截图
```

## 第 3 步：写第一批用户触达脚本（25–45 分钟）

### 社区帖模板（Reddit / Indie Hackers / 即刻）

**不是硬广**，是问问题：

```
Title: Anyone else struggle with [痛点]?

Body:
背景：我自己每天 X 次遇到 [问题]。
我现在的解决方案：[临时方案]，但它有 [缺陷]。
我做了个东西：[一句话讲产品]
TestFlight：[链接]

想听听大家：
- 你也遇到这个问题吗？
- 你现在怎么解决？
- 第一版你最想看到哪个功能？
```

### 短视频脚本模板（TikTok / Reels / 小红书）

```
[0-3s 钩子]
"你也每天午饭后困得不行？"

[3-10s 痛点共鸣]
"我也是。喝咖啡 30 分钟才起效，眯一会又怕睡过头。"

[10-25s 演示]
[屏幕录像：打开 App → 选 20min → 开始 → 闹钟响]

[25-30s CTA]
"叫 NapKit，链接在主页"
```

### KOL 私信模板

```
Hi [name],

我做了一个 [一句话讲产品]，看了你 [具体某条内容] 觉得你的受众可能用得上。

不是推广合作，单纯想让你试用一下：[TestFlight 链接]
如果你觉得有意思，能不能给我留个反馈？

不喜欢就忽略这条消息没关系。
```

**关键**：

- 不发模板群发，每条都提到对方某条具体内容
- 不一上来谈钱（KOL 烦透了）
- 准备好被无视，发 20 条能回 3 条算正常

## 第 4 步：设通过 / 失败标准（45–55 分钟）

### 不看下载数，看 3 个真指标

| 指标 | 通过线 | 失败处理 |
|---|---|---|
| **激活率**（下载 → 完成核心动作） | ≥ 30% | < 20% → 改 onboarding |
| **试用率**（激活 → 触发付费墙） | ≥ 40% | < 25% → 付费墙时机太早或太晚 |
| **付费率**（试用 → 真正付款） | ≥ 5% | < 3% → 价格或价值主张错了 |

### 30 天 = 60 / 100 / 200

第一个月的合理目标：

- **60 个真实激活用户**
- **100 个 TestFlight / App Store 下载**
- **200 个落地页访问 / 视频曝光**

不是数字越大越好，是这个**漏斗成立**。

### 什么时候停止 / 换方向

- 30 天后激活率 < 10% → 产品本身有问题，回 [第 3 篇](/zh/tutorials/app-mvp-swiftui/) 改 MVP
- 30 天后渠道没带来任何激活用户 → 换渠道（不是放弃产品）
- 30 天后激活率 OK 但付费率 0% → 进 [第 6 篇](/zh/tutorials/app-conversion-retention/) 改付费墙

## 第 5 步：今天就发出第一条（55–60 分钟）

**这一小时结束前，做出至少一个真实动作**：

- 发出第一条社区帖
- 上传第一条短视频
- 提交第一组广告创意
- 私信第一个 KOL

不要"等我都准备好了再开始"。准备永远没尽头，开始反馈才是真。
