> 系列总览：[App 出海指南](/zh/tutorials/app-playbook/)

新手做独立 App 最贵的错误，不是写代码慢，而是**先写三个月代码，再发现没人要**。

这篇教你用 60 分钟跑完一次真实的需求验证。验证通过再开始写代码；不通过就换方向，省下三个月。

## 🎯 你将完成

读完并动手做完，你会有：

- 一份 **30 条问题清单**（来自真实用户抱怨，不是你脑补）
- 一份 **10 条用户原话**（你以后写截图、广告、付费墙都靠它）
- 一个 **假 Web2App 验证漏斗**（落地页 → 假付费墙 → 留邮箱）
- 一份**通过 / 不通过**的判断：要不要继续做？

## 📋 前置要求

- 一个你**正在挠的痒**（自己每天遇到 3 次以上的小麻烦最佳）
- $20 左右的广告测试预算（不强制，但能极大加速验证）
- 一个能搭单页的工具账号：[Carrd](https://carrd.co/) / [Framer](https://framer.com/) / [Tally](https://tally.so/) 任选

---

## 第 1 步：挖痛点（0–15 分钟）

**核心心法**：不要从「我想做什么功能」开始，从「谁现在很痛、正在到处找办法」开始。

### 去这 4 个地方找

**1. App Store 找差评**

- 搜你方向的 3 个核心关键词（例如 "habit tracker"、"UV index"）
- 点开前 20 名 App
- 只看 **1–3 星差评**和**最近 3 个月的长评**
- 记录用户原话："我希望有…"、"为什么没有…"、"用了一周就…"

**2. Reddit 找抱怨**

- 在 r/sideproject、r/apps、r/&lt;你的细分领域&gt; 搜：
  - `app for [你的方向]`
  - `how do I [痛点]`
  - `alternative to [大厂产品]`
- 看高票回复里大家在抱怨什么，原话记下来

**3. TikTok / YouTube 评论区**

- 搜 "best app for X"、"X review"
- 看评论区里"有没有人推荐…"、"我一直找不到…"
- 这里的句子最接地气，写广告文案直接能用

**4. 自己的真实生活**

- 列出你**过去 7 天**遇到的所有小麻烦
- 不先想方案，只记问题
- 目标 30 条

### 工具加成（可选）

- **Astro / Appfigures / Sensor Tower**：看关键词搜索热度、竞品下载量
- **Google Trends**：确认是不是季节性热点，别把短期热度当长期市场
- **GummySearch**：聚合 Reddit 抱怨

<Checkpoint>
**这一步结束你应该有：**
- 30 条问题清单（用户原话，不要你二次加工）
- 至少 3 条以上重复出现的痛点
- 一个你**直觉觉得能做**的方向
</Checkpoint>

---

## 第 2 步：缩小方向（15–25 分钟）

**核心心法**：一个 App 只能先服务一个明确人群。人群混了，截图、文案、付费墙都会散掉。

### 用「人群 + 场景 + 结果」写一句定位

模板：

```
给 [谁]，在 [什么场景] 下，得到 [什么结果]。
```

**反例**（太泛）：

> 帮所有人记录日常习惯的 App

**好例**：

> 给做内容创作的人，在每天发完帖之后，自动整理本周数据并提醒下周选题的 App

### 把候选方向打分

挑出 3 个候选，按这 4 项打分（每项 1–5 分）：

| 维度 | 问自己 |
|---|---|
| **痛感强度** | 不用会损失时间 / 钱 / 健康 / 机会吗？ |
| **购买能力** | 用户已经为替代方案付过钱了吗？ |
| **获客路径** | ASO / TikTok / Reddit / Google Ads 里有清晰入口吗？ |
| **竞争缝隙** | 差评里有没有重复抱怨、而竞品长期没解决？ |

任何一项低于 3 分，**淘汰**。

### 不要这样做

- ❌ 同时服务两批相反需求的人群
- ❌ 为了"市场更大"把定位写成所有人都能用
- ❌ 用功能当卖点，要写**用户想要的结果**
- ❌ 因为关键词低难度就做—— Apple 现在会拒绝过于简单的工具 App

<Checkpoint>
**通过标准**：你能用 **15 秒**说清——
谁会搜它 / 为什么现在需要 / 竞品哪里没满足 / 你第一版只解决哪一步。
</Checkpoint>

---

## 第 3 步：搭假漏斗（25–45 分钟）

**核心心法**：不写一行 App 代码，先用一个网页验证有没有人愿意点、愿意留邮箱、愿意"付费"。

### 漏斗结构

```
广告 / 社区帖
   ↓
落地页 hero（一句话承诺）
   ↓
证明页（截图 / GIF / 用户原话）
   ↓
"假"付费墙（价格 + Start Free Trial 按钮）
   ↓
留邮箱页（"我们还在开发中，留下邮箱第一批通知你"）
```

### 用什么搭

最快的组合：

- **落地页**：Carrd ($19/年) 或 Framer 免费版
- **假付费墙**：同一个工具里加一段 section，写清 $4.99/月 / $29.99/年，下面放 Start Trial 按钮
- **邮箱收集**：Tally 表单 / Carrd 内置表单 / ConvertKit 免费版
- **追踪**：Plausible / Counterscale / GA4

### Hero 怎么写

3 行结构：

```
[一句话讲清楚你解决什么]
[副标题：给谁 + 在什么场景下用]
[CTA 按钮：Start 7-Day Free Trial]
```

**例子**（防晒提醒 App）：

```
Never get sunburned again.
A UV tracker that reminds you when to reapply sunscreen.
[ Start 7-Day Free Trial ]
```

### 假付费墙关键点

- 真实价格（$4.99/月 是 indie iOS App 起步常见档）
- 写清楚试用期长度、试用结束怎么扣费
- **不要收信用卡**、不要假装真扣费
- 点 Start Trial 之后跳到留邮箱页：
  > "Heads up! We're still polishing the app. Drop your email and you'll be among the first 100 to get free access."

### 埋点必装

- 落地页访问数
- 滚动到付费墙的比例
- 点击 Start Trial 按钮的比例
- 最终留邮箱的比例

<Checkpoint>
**这一步结束你应该有：**
一个可访问的 URL（例如 `your-idea.carrd.co`），从访问到留邮箱的完整漏斗能跑通。
</Checkpoint>

---

## 第 4 步：跑流量（45–55 分钟）

### 免费流量：5 个社区帖

**模板**（不是硬广，是问问题）：

```
标题：Anyone else struggle with [痛点]?

正文：
背景：我自己每天遇到 X 次 [问题]。
我现在的解决方案：[临时方案]，但它有 [缺陷]。
我做了个想法：[一句话讲产品]
预览：[落地页链接]

想听听大家：
- 你也遇到这个问题吗？
- 你现在怎么解决？
- 第一版你最想看到哪个功能？
```

发到 5 个相关社区：

- Reddit：r/sideproject + 你方向的细分 sub
- Indie Hackers
- Hacker News（Show HN，慎用，被踩很贵）
- X / Twitter（@相关 KOL，问反馈）
- 微信 / 即刻 / 小红书（如果做中文市场）

### 付费流量（可选但强烈推荐）

**Meta Ads $20 测 3 个角度**：

- 同一个落地页，做 3 套广告创意
- 每套创意只测**一个核心承诺**
- 跑 24–48 小时，每天预算 $5
- 关注：CTR、CTR → 付费墙、付费墙 → 留邮箱

### 不要做的事

- ❌ 一开始就跑大预算（看不出数据噪音）
- ❌ 同时测 5 个角度（数据稀释，啥都得不出）
- ❌ 用"震惊体"做广告（来的流量不精准）

---

## 第 5 步：判断（55–60 分钟）

### 通过标准

跑完小流量后，看这 3 个数：

| 指标 | 通过线 | 含义 |
|---|---|---|
| **付费墙到达率** | 12% – 25% | 访客愿意往下看 |
| **留邮箱率** | 2% – 5% | 有真实购买意向 |
| **访谈一致性** | ≥ 5 个人用同一句话描述同一个痛点 | 痛点是真的 |

3 项都过 → **继续**，进入下一篇：[App 细分定位](/zh/tutorials/app-niche-positioning/)

任何一项明显不达标 → **换方向**，回到第 1 步。

### 常见误判

- **下载意向高但不愿留邮箱** → 痛点没那么强，或者你的价格太高
- **CTR 高但付费墙到达率低** → 广告承诺和落地页脱节
- **留邮箱多但都不回复后续问卷** → 用户只是好奇，不是刚需

### 你应该花更多时间在这一步

> 验证阶段省一周，开发阶段亏三个月。

宁可在这里多投 $50 跑两轮广告，也不要凭感觉冲进开发。

<Checkpoint>
**60 分钟结束你应该有：**
- ✅ 30 条问题清单
- ✅ 10 条用户原话
- ✅ 1 个能跑通的假付费墙漏斗
- ✅ 一份明确的判断：**继续 or 换方向**

如果"继续"，下一篇我们把方向锁死、写成可执行的产品定位。
</Checkpoint>
