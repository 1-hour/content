## Step 1: Draw your own loop (0–10 min)

### Basic structure

Don't copy someone else's loop — draw yours from **your real data**:

```
  ┌──────────────────┐
  │ Main acquisition │
  └────────┬─────────┘
           │
           ▼
  ┌──────────────────┐
  │ Onboarding →     │
  │ Activation       │
  └────────┬─────────┘
           │
           ▼
  ┌──────────────────┐
  │ Trial → Payment  │
  └────────┬─────────┘
           │
           ▼
  ┌──────────────────┐
  │ Retention +      │ ──┐
  │ Reviews          │   │
  └────────┬─────────┘   │ feeds back
           │             │
           ▼             │
  ┌──────────────────┐   │
  │ Revenue reinvest │   │
  │ + Word of mouth  │ ──┘
  └──────────────────┘
```

### Fill in real numbers

For each node, write:

- Current value (% / users)
- Healthy baseline
- Gap

Example:

```
Main acquisition: 30 dl/day [TikTok]
Onboarding pass:  60% (baseline 70%)  ⚠️
Activation:       40% (baseline 50%)  ⚠️
Trial → paid:     8% (baseline 5%)    ✅
D7 retention:     22% (baseline 20%)  ✅
MRR:              $180
```

## Step 2: Bottleneck priority (10–25 min)

### Which to fix first

When multiple metrics are weak, fix in this order:

```
1. Activation (users never even reach value)
   ↓ then
2. Trial → paid (no money = no fuel)
   ↓ then
3. D7 retention (leaky bucket wastes new users)
   ↓ then
4. Acquisition cost (optimize last)
```

**Counter-intuitive**: many devs optimize acquisition first (more ads, more videos) but **with a leaky bucket, more traffic = more waste**.

### One fix at a time

Weekly experiment, max 2 parallel (and only if they don't affect the same segment).

### Optimizations not worth your time

- ❌ A/B testing icon color (< 1% impact)
- ❌ Optimizing app launch speed (unless > 3 seconds)
- ❌ Making UI "prettier" (users care about results, not aesthetics)
- ❌ Adding features to "please users" (more features = more risk, often worse retention)

## Step 3: Revenue reinvestment 50/30/20 (25–40 min)

### Split rule

Each month's net revenue (after Apple's 30%):

```
50% → back into the winning channel (more ads / more video / keyword tools)
30% → your salary (you need to live)
20% → tools / designer / edge experiments
```

### How to invest the 50%

**Not distributed evenly** — put it into **the channel where CAC < LTV is proven**.

Example:

- TikTok brings users, CAC ≈ $1, LTV ≈ $8 → **scale up**
- Meta ads CAC ≈ $4, LTV ≈ $5 → **hold**
- Reddit organic → **no $$, but post on schedule**

### When to open a new channel

Both conditions met:

1. Current main channel has **hit daily ceiling** (more budget → higher CAC)
2. Current main channel **revenue ≥ half of next month's target**

Example: TikTok = $300/mo, target $500 next month → TikTok must stabilize ≥ $250 before adding Reddit Ads.

### 20% edge experiments

Each month, 20% on high-risk-high-reward:

- New platforms (YouTube Shorts, Threads, new countries)
- New payment model (subscription → one-time + premium sub)
- Paid KOL collabs
- Beta features for small cohort

One-line log per experiment. **Kill it if no traction in 3 months.**

## Step 4: 6-month roadmap (40–55 min)

### Phased goals

```
M1: $100/mo - prove the loop (validation)
  - D7 retention ≥ 20%
  - Trial → paid ≥ 5%
  - Main channel ≥ 10 real users/day

M2-M3: $300-500/mo - fix the bucket (depth)
  - Fix worst 1-2 segments
  - 8-12 experiment logs
  - At least 1 piece of breakout content

M4-M6: $1k/mo - scale + new channel (expansion)
  - After main channel stabilizes, add second
  - Launch second-language market
  - Yearly subs > monthly subs share
```

### Different stages = different priorities

| Phase | Time allocation |
|---|---|
| M1 | 80% bucket-fixing + 20% content |
| M2-M3 | 60% bucket + 30% content + 10% new features |
| M4-M6 | 30% bucket + 40% new channels + 30% content |

### Don't do

- ❌ Second channel in M1 (first one isn't proven)
- ❌ Yearly plan in M1 (users don't trust you enough)
- ❌ Still tweaking onboarding in M4 (should be scaling)

### 3 signals to kill the app

If by month 4 all 3 are true → sunk cost, move on:

- MRR < $50 with no growth for 3 months
- D7 retention < 10%
- 3 new-channel experiments, all failed

Back to [Part 1](/en/tutorials/app-idea-validation/) for a new direction.

## Step 5: Weekly retro + next week (55–60 min)

### Weekly retro template

Every Sunday, 15 minutes:

```
Week ___ Retro

📊 Numbers
- MRR: $___
- New activated: ___
- D7 retention: ___%
- Trial → paid: ___%

🧪 This week's experiment
- Changed: ___
- Result: ___
- Decision (merge / revert / keep testing): ___

🎯 Next week
- Main bottleneck: ___
- Next experiment: ___
- Revenue reinvest: $___ → ___
```

### Where to write it

- A Notion "Weekly" page (easy to look back)
- Or post publicly on Twitter/X (build in public, generates impressions)
- Or IndieHackers post (deeper, gets you feedback)

Posting publicly turns the retro itself into content. The "retention feeds acquisition" arrow on your loop becomes naturally real.
