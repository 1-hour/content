到这里你应该有：能跑的 App + 至少一个稳定渠道 + 有付费用户 + 在做实验。

这一小时把所有动作**串成飞轮**。不再是修这修那，而是收入会自己滚雪球。

## 🎯 你将完成

- 一张**飞轮图**：你的获客 → 激活 → 付费 → 反哺
- **瓶颈优先级**：当多个数都不够好，先修哪个
- 一份**收入再投**规则：赚到的钱怎么回到获客
- 一个 **6 个月路线图**：从 $100/月 到 $1k/月

## 第 1 步：画你自己的飞轮（0–10 分钟）

### 基础结构

不要复制别人的飞轮——基于**你的真实数据**画：

```
  ┌─────────────┐
  │ 主渠道获客  │
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ Onboarding  │
  │ → 激活事件  │
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ 试用 → 付费 │
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ 留存 + 评分 │ ──┐
  └─────────────┘   │
         │          │ 反哺
         ▼          │
  ┌─────────────┐   │
  │ 收入再投 +  │   │
  │ 口碑素材    │ ──┘
  └─────────────┘
```

### 填上你的真实数据

每个节点写：

- 现在的数值（百分比 / 用户数）
- 健康基线
- 跟基线的差距

例子：

```
主渠道获客: 30 下载/天 [TikTok]
Onboarding 通过: 60% (基线 70%)  ⚠️
激活事件: 40% (基线 50%)         ⚠️
试用 → 付费: 8% (基线 5%)        ✅
D7 留存: 22% (基线 20%)          ✅
月收入: $180
```

## 第 2 步：瓶颈优先级（10–25 分钟）

### 哪段先修

当多段数据都不达标时，按这个顺序修：

```
1. 激活事件（用户根本没用上）
   ↓ 修好了再往下
2. 试用 → 付费（不付钱再多用户也没收入）
   ↓ 修好了再往下
3. D7 留存（用户跑了再多新增也是漏桶）
   ↓ 修好了再往下
4. 获客成本（这是最后一步优化）
```

**反逻辑**：很多人先去优化获客（投更多广告、做更多视频），但**底下漏桶没补，流量越大越浪费**。

### 一次只修一个

每周 1 个实验，最多并行 2 个（且 2 个互不影响段）。

### 不要做的优化

- ❌ A/B 测图标颜色（影响 < 1%）
- ❌ 优化 App 启动速度（除非真的 > 3 秒）
- ❌ 改 UI 让"更好看"（用户不在乎好看，在乎能解决问题）
- ❌ 加功能"让用户更喜欢"（加功能 = 风险 + 留存可能反而下降）

## 第 3 步：收入再投 50/30/20（25–40 分钟）

### 切分规则

每月真实收入（扣除 Apple 30%）按这个比例分：

```
50% → 回到胜出渠道（投广告 / 拍更多视频 / 关键词工具）
30% → 自己工资（你也要活）
20% → 工具 / 设计师 / 边缘实验
```

### 50% 怎么投回获客

**不是均摊在所有渠道**——投到**已经证明 CAC < LTV 的那个**。

例子：

- TikTok 渠道带来用户，CAC ≈ $1，LTV ≈ $8 → **加预算**
- Meta 广告 CAC ≈ $4，LTV ≈ $5 → **保持**
- Reddit 自然帖 → **不投钱，但每周固定时间发**

### 什么时候再开新渠道

满足这 2 个条件：

1. 当前主渠道**单日已增长到瓶颈**（再加预算 CAC 上升）
2. 当前主渠道**收入 ≥ 你下个月一半目标**

例子：TikTok 当前每月 $300，下个月想做到 $500 → TikTok 至少能稳 $250 才考虑加 Reddit Ads。

### 20% 边缘实验

每个月留 20% 做"高风险高回报"实验：

- 新平台测试（YouTube Shorts、Threads、新国家）
- 新付费模式（订阅 → 一次性 + 高级订阅）
- KOL 付费合作
- 新功能小灰度

记一行日志，**3 个月没起来就停**。

## 第 4 步：6 个月路线图（40–55 分钟）

### 阶段性目标

```
M1: $100/月 - 跑通飞轮（验证）
  - 留存 D7 ≥ 20%
  - 试用 → 付费 ≥ 5%
  - 主渠道单日 ≥ 10 个真实用户

M2-M3: $300-500/月 - 优化漏桶（深耕）
  - 修最差的 1-2 段
  - 实验日志记 8-12 条
  - 至少 1 篇内容 / 视频破圈

M4-M6: $1k/月 - 加速 + 开新渠道（扩张）
  - 主渠道收入稳定后加第二渠道
  - 第二语言市场上线
  - 一年订阅占比 > 月订
```

### 不同阶段做不同的事

| 阶段 | 时间分配 |
|---|---|
| M1 | 80% 修漏桶 + 20% 内容 |
| M2-M3 | 60% 修漏桶 + 30% 内容 + 10% 新功能 |
| M4-M6 | 30% 修漏桶 + 40% 新渠道 + 30% 内容 |

### 不要这样做

- ❌ M1 就想做第二渠道（你都没跑通第一个）
- ❌ M1 就上一年订阅（用户对你信任不够）
- ❌ M4 还在反复调 onboarding（这时候应该规模化）

### 决定停掉的 3 个信号

到第 4 个月还满足这 3 条 → 这个 App 不行了，沉没成本止损：

- 月收入 < $50 且 3 个月没增长
- D7 留存 < 10%
- 新渠道实验 3 个都没起来

回到 [第 1 篇](/zh/tutorials/app-idea-validation/) 找新方向。

---

## 第 5 步：本周复盘 + 下周动作（55–60 分钟）

### 每周复盘模板

每周日花 15 分钟写：

```
Week ___ 复盘

📊 数据
- 月收入: $___
- 新激活用户: ___
- D7 留存: ___%
- 试用 → 付费: ___%

🧪 本周实验
- 改了什么: ___
- 结果: ___
- 决策（合并 / 回滚 / 继续测）: ___

🎯 下周
- 主要瓶颈: ___
- 下个实验: ___
- 收入再投: $___ → ___
```

### 写到哪

- Notion 一个"周报"页（方便回看）
- 或 Twitter/X 公开发（build in public，能带来曝光）
- 或 IndieHackers post（更深度，能拿到反馈）

公开发的好处：复盘本身就是内容素材，飞轮里"留存反哺获客"这一环天然成立。

## 🏁 系列结语

你做完了 7 篇。但**真正的工作刚开始**：

- 验证 → 定位 → MVP → 上架 → 冷启动 → 转化 → 飞轮
- 这 7 步**每一步都会被现实打脸**，不要紧，回来看对应那篇

独立 App 不靠灵感，靠**重复**。靠每周 1 个实验、靠每月 1 个复盘、靠每年 1 次大的方向校准。

**祝你的 App 长成。**
